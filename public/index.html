Tại Sao Nên Dùng Visual Regression Testing?

Phát hiện lỗi giao diện trước khi người dùng nhìn thấy — tự động hóa hoàn toàn việc kiểm tra pixel.

📸

Screenshot Testing

Kiểm tra giao diện website có thay đổi không bằng cách so sánh ảnh chụp màn hình tự động.

🔍

Pixel Perfect

So sánh từng pixel giữa 2 ảnh với threshold tùy chỉnh, phát hiện thay đổi nhỏ nhất.

🌐

Multi-Browser

Chạy test đồng thời trên Chromium, Firefox và WebKit để đảm bảo tương thích đa trình duyệt.

📊

HTML Report

Xem kết quả test chi tiết với diff image, so sánh expected vs actual ngay trên trình duyệt.

⚡

Snapshot Testing

Lưu snapshot lần đầu, tự động so sánh các lần sau. Workflow đơn giản, hiệu quả.

🔄

Diff Image

Tạo ảnh diff tô đỏ phần khác biệt, giúp xác định chính xác vùng thay đổi trên giao diện.

📖 Hướng Dẫn

Cách Sử Dụng Dự Án

Từ khởi động server đến xem báo cáo — làm theo các bước dưới đây để bắt đầu với Visual Regression Testing.

1

🏁 Chạy Demo Website

Khởi động HTTP server local để serve trang web demo tại http://localhost:3000

📋 Sao chép npm run server
2

🧪 Test So Sánh Ảnh PNG

So sánh 2 ảnh PNG từ thư mục images/expected và images/actual. Kết quả diff được lưu vào images/diff.

📋 Sao chép # So sánh tất cả ảnh trong thư mục
npm run test:compare-images
💡 Đặt ảnh chuẩn trong images/expected/ và ảnh cần kiểm tra trong images/actual/.
3

🌍 Test So Sánh 2 Website

Chụp ảnh 2 trang web khác nhau rồi so sánh pixel. Mặc định so sánh playwright.dev với chính nó (sẽ PASS). Sửa URL trong file test để tạo lỗi.

📋 Sao chép # Chạy test so sánh website
npm run test:compare-websites
💡 Mở tests/compare-websites.spec.js dòng 24, đổi URL để tạo visual regression.
Ví dụ: await page.goto('https://playwright.dev/docs/intro');
4

📸 Test Snapshot Visual

So sánh full page screenshot với snapshot đã lưu trước đó. Lần đầu: tạo snapshot mới → PASS. Lần sau: so sánh → PASS nếu giống, FAIL nếu khác.

📋 Sao chép # Chạy test snapshot
npm run test:visual
💡 Để cập nhật snapshot mới: npx playwright test --update-snapshots
5

📋 Test Demo Website

Chạy bộ 7 test cho trang demo: chụp full page, header, cards, features, responsive mobile (375×812), hover effect, và kiểm tra DOM structure.

📋 Sao chép # Chạy test demo
npm run test:visual
💡 Sửa public/index.html (ví dụ đổi gradient) rồi chạy lại test để thấy visual regression FAIL.
6

📊 Xem HTML Report

Sau khi test chạy xong, mở HTML report để xem chi tiết từng test case, ảnh expected vs actual, và diff image.

📋 Sao chép # Mở report
npm run report
💡 Trong report, click vào test FAIL để xem ảnh diff với phần khác biệt được tô đỏ.
